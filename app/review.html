Portfolio Review

I have reviewed my portfolio multiple times with my tutors, classmates and industry professionals. Some key usability issues were found during the tests. I have created a list of issues and solutions:

Issue #1: The main page were too long, and the navigation triggered only if the user scrolled down on the page to a certain point, thus some people did not notice the navigation bar and have not seen the rest of the pages.

Solution #1: Some users thought my website consists only one page, because of the hidden navigation. It made me worried as I have written a loads of content. To fix this I have made the navigation bar prominent and visible, I also created an about page to make the main page a bit shorter. Now it is much easier to navigate and find the content on the website.

Issue #2: The header background on some pages was a landscape image, and it made the text difficult to read.

Solution #2: The background image has been changed to a pattern image, which makes good contrast, and also smaller file size to download.

Issue #3: Weak Meta description. “Krisztian Pinter's Agency Ready Web Developer Portfolio.” It did not reflect to my skills.

Solution #3: Changed the meta description: “Looking for a skilled UX designer & Front-End Developer? Look no further, contact me.” It is important to have an engaging meta description, because that’s what people can see on Google, when they search. It has to convince people to click on my website and not on others.

Issue #4: Due to the font-family and the small letter-spacing, it was difficult to read the body text and it became boring after a while.

Solution #4: I’ve been inspired by Jason Santa Maria – On Web Typography book, and realized, that the fonts I was using were not appropriate for long text, and I set too small letter-spacing which also affected the readability. Thus I have changed the paragraphs to a well-known Serif font, called EB Garamond and also set the letter-spacing to 0.05em. Probably when you read this text, you already perceived it is much more comfortable for your eyes.

Issue #5: The contact form was not accessible.

Solution #5: When I firstly built this portfolio, the contact form had only some placeholder text. I thought it is perfectly enough. Unfortunately the screen readers skip the placeholders and it makes impossible to understand the website for disabled users. Then I read a webpage about Form Accessibility (http://webaim.org/techniques/forms/). It suggests to use labels, as it makes the form well-organized and highly usable. Some users were missing my downloadable CV and my email address from the contact section, as not everyone like to send email through forms, thus I have added these features on the page.
