Performance optimisation in web design

I have recently read the Responsive Report 2014 by Gridset. It has highlighted the key concerns and challenges that the web industry facing and shown the direction where it goes. The report compares the previous three years, built on the answers given by people that are working in this field. It was interesting to see that the major challenge in 2014 was the website performance optimisation. Therefore, I have decided to write a blog post upon this topic.

Why do we need to optimise websites?

The loading speed of websites has become more important than the actual design. The reason behind this that nowadays more internet users are visiting websites from mobile devices than desktop computers. Due to the speed and bandwidth limits of mobile broadband, developers and designers has to compromise and optimise their assets.

Did you know that one in four visitors would abandon a website if it does not load within 4 seconds? Furthermore, 46% of them do not revisit poorly performing websites.

When Mozilla increased page speed by 2.2 seconds, their download figures rose by 15.4%. It is 10 million downloads a year!

Let’s take an ecommerce example: By every 1 second delay in page load time would lead $1.6 billion yearly losses for Amazon.

You can see from the above examples that there is a relation between load speed and conversion. Do not make your users frustrated with a slow website, unless you want to achieve poor results.

What can I do?

Firstly, I would recommend testing your current speed with different online tools. I am mostly using Webpagetest and Google Pagespeed, but there are many different tools out there, such as GTMetrix or Pingdom .

After you have received and analysed your data, most of the time it turns out that 80-90% of the issues happen in the front-end. I have created a basic, small list below about how to prevent and/or solve the speed issues:

Scripts and stylesheets:

If you have multiple scripts and stylesheets on your website, it could make a negative impact on your website’s performance. Solution: Combine your stylesheets, script files and minify them. (Here is a handy tool for this task: http://refresh-sf.com/ ) Then place your script file on the bottom of the page, in order to load it last.

Images:

Most of the time your images has the biggest weight in the total size of your project, thus it has key importance to optimise your pictures if you want a lean website.

Solution: Firstly, you have to decide the correct file format. (eg. If you are not using the transparent feature of the .png file format, that what you really need is a much smaller .JPG file. )

Resize your image assets, using Photoshop or similar photo-editor tool, and then reduce the quality. Unfortunately, it is coming with loads of compromises; probably it is the biggest nightmare of designers. However, do not forget: Speed is more important!

If you want to read about this topic in more depth, I would recommend the following Smashing magazine articles:

http://www.smashingmagazine.com/2009/07/clever-jpeg-optimization-techniques/

http://www.smashingmagazine.com/2009/07/clever-png-optimization-techniques/

Image Sprites:

One way to reduce HTTP request is to create an image sprite. It is a method to merge all of your image assets into one file. The benefit of this, that as soon as your main image is loaded, all of your images will be available. There is a super useful tool created by Google, called SpriteME . It generates the sprite and creates all the necessary CSS for it. All you need to do is just run it on your website.

Online image optimisation tools:

Kraken.io & Smush.it

When using the “Save for web” option in Photoshop, there are still remaining ‘extra’ data in your images. The above tools can help you to remove these extra information bits and further reduce the file-size without sacrificing the quality of your images.

Lazy Load:

Lazy load delays loading of images in long web pages. Images outside the viewport are not loaded until the user scrolls to them. Using this clever script can result faster page load.

http://www.appelsiini.net/projects/lazyload

Content Delivery Networks:

If you use a library or CSS provided by a library, make sure to use the hosted versions of the files (such as jQuery or Bootstrap)

Key benefits of using CDN:

  • Visitors to your website will get the JavaScript and CSS from the server that is as geographically close to them as possible and won’t have to wait for your server to send the information from around the globe.
  • There is a high probability that these servers are faster than yours.
  • Visitors who have visited other websites that use the same includes will already have them on their computers and won’t need to load them again.
  • You save on bandwidth and can easily upgrade the library by changing the version number of the include.

I hope by now you understood why performance optimisation is important, and you are able to apply these basic methods on your own projects to make a better user experience.
